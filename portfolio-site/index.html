소개

복잡함보다 실용성을, 새로움보다 필요함을 추구합니다.

많은 사람들이 쉽게 접근하고 자주 사용하는 서비스를 만드는 데 가치를 두며, 작지만 확실한 편리함을 제공하는 데 집중합니다.

여러 동아리 부장 경험으로 협업과 소통이 능숙합니다.

개발 철학

  • 필요한 가치를 빠르게 검증하고 작게라도 배포합니다(Ship small, ship often).
  • 초기 기능보다 사용성·안정성을 우선합니다(초심자 친화, 오류 복구 용이).
  • 팀이 함께 유지·확장할 수 있는 구조를 선호합니다(명확한 책임, 단순한 인터페이스).

문제 해결 사례

  • [마루포트] 캔버스 객체 드래그 시 축 고정(Shift) 동작이 튀는 문제를 로직 분리와 스냅 우선순위 재설계로 해결 — 사용자 테스트에서 오동작 제보 0건.
  • [마루포트] Undo/Redo 수행 시 상태 이탈 문제를 불변성 보장 + 배치 업데이트로 개선 — 연속 작업 시 프레임 드랍 완화.
  • [AI더버] 낙상/온열 탐지 민감도 편차를 전처리 파이프라인(이상치 제거, 이동평균)으로 안정화 — 시뮬레이션 정확도 향상.

숫자로 보는 영향

  • 해커톤 대상 1회, 금상 2회(교내/외)
  • 팀 리드 경험 3건(동아리/프로젝트)
  • 웹 기반 에디팅 UX 실험 10+ 차례

핵심 역량

  • 프론트엔드 아키텍처 설계와 상태 관리(Zustand 등) 경험
  • DOM 기반 캔버스 에디터 구현(드래그·리사이즈·스냅, 히스토리/Undo·Redo)
  • 문서 출력 흐름 구축(PDF 내보내기, 미리보기, 템플릿 시스템)
  • 리더십·커뮤니케이션(여러 동아리/프로젝트 팀장 경험)

관심 분야

  • 생산성 도구와 교육·생활 밀착형 서비스
  • 웹 기반 에디팅/디자인 툴, 인터랙션 최적화
  • AI 보조 기능을 실사용 플로우에 자연스럽게 녹이는 UX
